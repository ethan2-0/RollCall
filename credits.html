RollCall is the cullimination of a lot of work by the free software community.
We just put it together.

Here are all of the projects that RollCall depends on. Without these, this service would be impossible. Bootstrap Bootstrap is a UI framework, and it's just awesome. I don't think I'll ever develop a website without it ever again; the grid system is especially useful. We use it mostly for its grid system, buttons, and dialogs - but it's used in a lot of other places, too.
Official Website | GitHub jQuery jQuery makes DOM interaction and AJAX so much easier, thus making pretty much all the UI code much easier to write. Literally everything you see in the RollCall webapp is put there by jQuery.
Official Website | GitHub Firebase All the data is stored in Firebase. Not only that, I don't have to remember the syntax of SQL, or worry about query injection vulnerabilities, or even worry about the server at all - they handle it.
Official Website PeerJS PeerJS handles all of the calls. Every second of call you make is because of PeerJS. I don't think I need to say more.
Official Website | GitHub (client source) | GitHub (server source) Velocity.js Velocity.js does almost all of the animations. It's easy to use, and it's really smooth - always smoother than jQuery animations, and usually smoother than CSS transitions.
Official Website | GitHub Bootbox.js All of the modal dialogs are handled by Bootbox.js. Although I could have done them by hand with Bootstrap, Bootbox makes it much, much easier and shorter.
Official Website | GitHub CSS3 & HTML5 Literally, I'm thanking the internet here - this entire website is possible only due to the HTML5 effort and CSS3. HTML5 makes HTML so much more powerful. WebRTC WebRTC is the underlying framework behind PeerJS. It's an effort to enable real-time communications, or RTC on the web, hence WebRTC. It is kind of an odd specification in that it leaves many areas undefined, but with a good library, it's really powerful.
